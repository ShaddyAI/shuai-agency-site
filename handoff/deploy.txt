VERCEL DEPLOYMENT
-----------------
To deploy to Vercel:

1. Install Vercel CLI:
   npm i -g vercel

2. Login:
   vercel login

3. Deploy to staging:
   vercel

4. Deploy to production:
   vercel --prod

Preview URL (staging): 
Will be generated after first deployment

Production URL:
https://yourdomain.com (after custom domain setup)

ENVIRONMENT VARIABLES REQUIRED
-------------------------------
Set these in Vercel dashboard before deploying:

✓ NEXT_PUBLIC_SITE_URL
✓ DATABASE_URL
✓ REDIS_URL  
✓ OPENAI_API_KEY
✓ ADMIN_PASSWORD_HASH
✓ GHL_API_KEY (optional, will use mock if not set)
✓ GHL_ACCOUNT_ID (optional)
✓ N8N_WEBHOOK_SECRET (optional)

See .env.example for full list

DATABASE SETUP
--------------
1. Provision Postgres with pgvector support:
   - Vercel Postgres (recommended)
   - Supabase
   - Neon
   - Self-hosted with pgvector extension

2. Run migrations:
   npm run db:migrate

3. Seed data:
   npm run db:seed

4. Index embeddings:
   npm run index:embeddings

REDIS SETUP
-----------
Provision Redis instance:
- Upstash (recommended for Vercel)
- Vercel KV
- Self-hosted

-------------
If issues occur after deployment:

1. Revert to previous deployment in Vercel:

2. Or redeploy previous commit:
